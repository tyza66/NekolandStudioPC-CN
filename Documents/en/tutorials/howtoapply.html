How to apply the script

Apply directly to a file

This method is used when the script needs a script that should run when the server is started or when the client is turned on.

First, the server script is in the /ServerScripts folder under the project folder, Client scripts are applied by adding *.lua files to the /Scripts folder under the project folder.

EX) Create a file after creating a file called Hello.lua under the /ServerScripts folder

Apply to events

This method is used when you need a script to run when the unit touches an event.

First, create an event unit, apply the'script' event to the unit, and write the necessary script to apply.

When executing a script on an event unit, the unit and event keywords are added to the local table.

unit represents the player unit that fired this event, and event represents the event unit. However, the unit keyword cannot be used if there is no unit that triggered the event, such as automatic start, when the event start condition is a map.

EX) Add a'script' event to the event unit that starts when the button is pressed.
Enter the unit.SendCenterLabel("Hello!") script

Using the /script command

This method is used when you simply want to test the script. /script [Script content] If you enter it into the chat window, the script content is executed.

If you run the script in that way, you can use the me and unit keywords added to the local table. me is the player who entered the command and unit is the player unit that entered the command.

EX) /script unit.SendCenterLabel("Hello!")

You have administrator privileges for the game or can only use it in a test environment.

Script in Damage Formula

In the database, there is a space for entering damage formulas such as skills and status, and the damage formulas written here are also executed as scripts.

Local tables in the formula include a, b, critical, level, etc.

a is the attacker, b is the defender, critical is critical or not, and level is the skill level. The damage formula used in database-state is only a and b in the local table

EX) a.atk-b.def

Information

The unit keyword exists in the local table in the space where you enter the script to find the player unit that is usually the subject.

Variables created in global space are shared for all script executions.

It is also possible to define a function in a script that is applied to a file and use the function in a script when an event is executed.

Server scripts and client scripts have different uses and usages, and even scripts executed within the same event may cause errors depending on the server and client. All of the examples in this document use server scripts as examples.

EX) Enter the server script in the client script field
